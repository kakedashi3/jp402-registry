AIエージェントが探して買う時代、売り手の問題は「広告を見てもらう」ではなく 「エージェントに発見・選別してもらう」に変わる。 jp402-registry は、準拠すれば自動で載る公開の地図。 中央台帳に頼らず、JPYC/Polygon の x402 経済を発見可能にする。

ディスカバリーレイヤーとは

「載る」と「信頼される」を分けた、公開の発見層

検索が x402 を Polygon ごと外して空白になった領域を、中央集権の台帳ではなく 準拠ベースの自前ループで埋める。誰の許可も要らず、けれど信頼は可視化する。

🗺️

載る = permissionless

自ドメインに準拠カタログ（.well-known）を置き、URL を1行 PR すれば自動で載る。運営が登録可否を決めない。

🛡️

信頼 = 別レイヤー

詐欺も載れる。だから弾かず、着金実績・宛先一致・準拠度などの信頼シグナルを透明に可視化。載る≠勝つ。

🔌

標準準拠で孤立しない

x402 Bazaar Discovery に準拠し、JP 固有項目だけ x-jp402 区画に隔離。汎用クローラは標準部だけ読める。

コードでできる層は、もう AI が書ける。決済を確定するファシリテーターも、AI に決済させる仕組みも。 でもディスカバリーだけは AI に作れない——その価値は「どの売り手が、載ることに同意し、どれだけ信頼が積まれているか」で、 レイヤーと売り手の合意でしか構成できないから。だから今やるのは儲けることではなく、 飽和が来る前に信頼と標準の旗を立てておくこと。
